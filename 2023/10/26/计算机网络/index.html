计算机网络

发表于2023-10-26|更新于2023-11-08|计算机
|阅读量:

第一章 概述

1. 因特网概述

1.1 网络、互联网、因特网

  • 网络：若干结点和连接这些结点的链路组成。
  • 互联网：多个网络通过路由器互连起来。网络的网络
  • 因特网：世界上最大的互联网

因特网服务提供者ISP

1.2 因特网的组成

  • 边缘部分：由所有连接在因特网上的主机组成。这部分是用户直接使用的，用来进行通信和资源共享。
  • 核心部分：由大量网络和连接这些网络的路由器组成。这部分是为边缘部分提供服务的（提供连通性和交换）。

路由器是实现分组交换的关键构件，其任务是转发收到的分组

2. 交换方式

三种交互方式：电路交换、分组交换、报文交换

2.1 电路交换

  • 电话交换机接通电话线的方式称为电路交换
  • 三个步骤
    1. 建立连接（分配通信资源）
    2. 通话（一直占用通信资源）
    3. 释放连接（归还通信资源）
  • 优点：通信时延小、有序传输、没有冲突、适用范围广、实时性强、控制简单
  • 缺点：建立连接时间长、线路独占，使用效率低、灵活性差、难以规格化

    当使用电路交换来传输计算机数据，其线路的传输效率往往很低。【突发】通常使用分组交换

2.2 分组交换

  • 发送方：构造分组、发送分组
  • 路由器：缓存分组、转发分组 即存储转发
  • 接收方：接受分组、还原报文

优点：无需建立连接、线路利用率高、简化存储管理（长度固定）、加速传输、减少出错概率和重发数据量
缺点：引起转发时延、需要传输额外的信息量、

2.3 报文交换

报文大小没有限制，要求交换结点有较大的缓存空间。
主要用于早期的电报通信网

优点：无需建立连接、动态分配线路、提高线路可靠性、提高线路利用率、提供多目标服务、
缺点：引起了转发时延（交换机存储转发）、需要较大的存储缓存空间、需要传输额外的信息量、对于数据报服务，存在失序、丢失或重复分组的问题；对于虚电路服务，存在呼叫建立、数据传输和虚电路释放三个过程
三者的对比

3. 计算机网络的定义和分类

3.1 定义

  • 互连：有线或无线方式进行数据通信
  • 自治：独立的计算机，有自己的软件和硬件，可单独运行
  • 集合：至少两台计算机

3.2 分类

  1. 按交换技术分类
  2. 按使用者分类：公用、专用
  3. 按传输介质分类：有线、无线
  4. 按覆盖范围分类：广域网WAM、城域网MAN、局域网LAN、个域网PAN
  5. 按拓扑结构分类：总线型网路、星型网络、环形网络、网状型网络

4. 性能指标

速率、带宽、吞吐量、时延、时延带宽积、往返时间、利用率、丢包率

4.1 速率

比特：数据量单位，1或0。1字节（Byte）=8比特（bit）
速率：连接在计算机网络上的主机在数字信道上次送比特的速率，比特率、数据率

4.2带宽

4.3吞吐量

单位时间内通过某个网络的数据量
受网络带宽或额度速率的限制

4.4时延

网络时延=发送时延+传播时延+处理时延
发送时延=分组长度(b)/发送速率(b/s)
传播时延=信道长度(m)/电磁波传播速率(m/s)
处理时延一般不计算

4.5时延带宽积

时延带宽积=传播时延 X 带宽
理解：在发送端发送的第一个比特到达终点时，发送端就已经发送了时延带宽积个比特的数据

4.6往返时间

RTT： 发送端发送到发送端接受到确认发送消息的时间。

4.7利用率

4.8丢包率

  • 定义：一定时间内丢失的分组数量与总分组数量的比率
  • 分组丢失的情况：
    • 在传输过程中出现误码
    • 到达队列已满的分组交换机时被丢弃，网络堵塞
  • 网络堵塞情况：
    • 无堵塞：丢包率0
    • 轻度堵塞：丢包率1%-4%
    • 严重堵塞：丢包率5%-15%

5.1 常见的计算机网络体系结构

  • OSI体系结构：七层
  • TCP/IP体系结构：四层

5.2 分层的必要性


#### 5.3 专业术语
- 实体：任何可发送或接收信息的硬件或软件进程（例如网卡和浏览器进程）
- 对等实体：收发双方相同层次中的实体（例如双方的网卡）
- 协议：控制两个对等实体进行逻辑通信的规则的集合
- 协议的三要素
- 语法：定义所交换信息的格式
- 语义：定义收发双方要完成的操作
- 同步：定义双方的时序关系
- 服务：在协议的控制下，两个对等实体间的逻辑通信使得本层能够向上一层提供服务
- 要实现本层协议，需要使用下面一层提供的服务
- 协议是水平的，服务是垂直的
- 下面的协议对上面的实体是透明的（不可见）
- 服务访问点：在同一系统中相邻两层的实体交换信息的逻辑接口，用于区分不同的数据类型
- 服务原语：上层使用下层服务时与下层交换的命令
- 协议数据单元PDU：对等层次之间传送的数据包
- 服务数据单元SDU：同一系统内，层与层交换的数据包

第二章 物理层

1. 物理层的基本概念

  • 物理层考虑的是怎样才能在传输媒体上传输数据比特流
  • 物理层为数据链路层屏蔽各种传输媒体的差异
  • 物理层协议的主要任务：
    • 机械特性：指明接口所用接线器的物理信息。(形状、尺寸)
    • 电气特性：接口电缆电压范围
    • 功能特性：电压代表的意义
    • 过程特性：事件的出现顺序

2. 物理层下的传输媒体

  • 导引型传输媒体
    • 同轴电缆
    • 双绞线
      • 绞合的目的：减少内部干扰和外部干扰
    • 光纤
      • 双模光纤
      • 单模光纤
    • 电力线
  • 非导引型传输媒体
    • 无线电波
    • 微波

3. 传输方式

  • 串行传输：例如计算机网络，一根传输线
  • 并行传输
  • 同步传输
    • 外同步：添加一条单独的时钟信号线
    • 内同步：发送端将时钟同步信号编码到发送数据中一起传输（曼彻斯特）
  • 异步传输
    • 字节之间异步（字节间时间间隔不固定
    • 字节中的每个比特仍要同步（各比特持续时间相同
  • 单向通信（单工）
  • 双向交替通信（半双工）
  • 双向通信（全双工）

4. 编码与调制

数据通信常用术语

  • 消息：需要计算机处理的文字、图片、音频以及视频等。
  • 数据：运送消息的实体，计算机只能处理二进制数据。
  • 信号：数据的电磁表现
    • 基带信号：来自信源的原始电信号
      • 数字基带信号：在计算机内部，cpu与内存之间传输的信号
      • 模拟基带信号：例如，麦克风采集到声音之后产生的音频信号
  • 编码：
    • 码元：在使用时间域的波形表示数字信号时，代表不同离散值数值的基本波形。
    • 常用编码：
      • 不归零编码：存在同步问题
      • 归零编码：自同步，编码效率低
      • 曼彻斯特编码：传统以太网，10Mb/s，中间时刻发生变化代表数据
      • 差分曼彻斯特码：变化少，更适合较高的传输速率，开始处电平发生变化代表数据
