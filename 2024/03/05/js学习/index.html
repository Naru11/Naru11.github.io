day1

  1. 为什么const声明的对象可以修改里面的属性?

    因为对象是引用类型，里面存储的是地址，只要地址不变，就不会报错建议数组和对象使用 const 来声明

  2. 什么时候使用let声明变量?

    如果基本数据类型的值或者引用类型的地址发生变化的时候，需要用let。比如 一个变量进行加减运算，比如for循环中的 i++

  3. innerText和innerHTML的区别：

    前者文本中包含的标签不会被解析后者会。

  4. 控制样式属性

  • css 属性的 - 连接符与 JavaScript 的 减运算符
    冲突，所以要改成驼峰法
    box.style.backgroundColor = ‘pink’

  • 操作类名(className) 操作CSS,为了解决className 容易覆盖以前的类名，我们可以通过classList方式追加和删除类名
    box.className = ‘pink’

  1. addEventListener 是 DOM 对象专门用来添加事件监听的方法，它的两个参数分别为【事件类型】和【事件回调】。addEventListener 第3个参数为 true 表示捕获阶段触发，false 表示冒泡阶段触发，默认值为 false

  2. 事件类型

  • 鼠标事件：mouseenter
  • 键盘事件：keydown keyup
  • 焦点事件：focus 获得焦点 blur 失去焦点
  • 文本框输入事件:input
  1. 事件对象 事件对象中包含了哪些有用的信息：
  • ev.type 当前事件的类型
  • ev.clientX/Y 光标相对浏览器窗口的位置
  • ev.offsetX/Y 光标相于当前 DOM 元素的位置
  1. 环境对象
  • 环境对象指的是函数内部特殊的变量 this ，它代表着当前函数运行时所处的环境。
  • 【谁调用 this 就是谁】是判断 this 值的粗略规则
  • 箭头函数没有this
  1. 事件流
  • 任意事件被触发时总会经历两个阶段：【捕获阶段】和【冒泡阶段】。

简言之，捕获阶段是【从父到子】的传导过程，冒泡阶段是【从子向父】的传导过程。

  1. 阻止冒泡：阻止捕获和冒泡
    阻止冒泡是指阻断事件的流动，保证事件只在当前元素被执行，而不再去影响到其对应的祖先元素。

  2. 事件解绑:比如点击一次之后不能再次点击
    btn.removeEventListener(‘click’,fn)

  3. 事件委托:多个子元素的相同事件由父元素执行

  • 利用事件流的特征，可以对上述的代码进行优化，事件的的冒泡模式总是会将事件流向其父元素的，如果父元素监听了相同的事件类型，那么父元素的事件就会被触发并执行，正是利用这一特征对上述代码进行优化。
  • 获取具体的子元素：e.target.tagName
  1. 阻止元素默认行为:比如表单提交

e.preventDefault()

  1. 页面加载事件:
    加载外部资源（如图片、外联CSS和JavaScript等）加载完毕时触发的事件。有些时候需要等页面资源全部处理完了做一些事情
  • 事件名：load 监听页面所有资源加载完毕：
  • DOMContentLoaded dom页面加载完时开始（不用等图片等其他资源）docuement.~
  1. 页面滚动事件：
  • 滚动条在滚动的时候持续触发的事件
  • 获取滚动位置 scrollLeft scrollRight：溢出的大小
  • 检测页面滚动的头部距离：document.documentElement.scrollTop

  1. DOM 节点
    掌握元素节点创建、复制、插入、删除等操作的方法，能够依据元素节点的结构关系查找节点。
  • 父节点：node.parentNode，可以依次往上查找
  • 子节点：
    • childNodes，获取全部的子节点，回车换行会被认为是空白文本节点

  • 兄弟节点：
    • 上一个兄弟：li.previousElementSibling
    • 下一个：li.nextElementSibling
  1. 增加节点
  • 创建节点：const p = document.createElement('p')
  • 追加节点到父元素的最后一个子元素：document.appendChild(p)
  • 追加节点到父元素的某个子元素前: ul.insertBefore(li1, ul.children[0])
  • 克隆节点：const li1=父元素.children[0].cloneNode(true),true是深克隆，会克隆整个元素，否则只克隆标签
  1. 删除节点

  1. M端事件（移动端）
  • touchstart：开始触摸
  • touchmove：移动
  • touchend：停止触摸
  1. BOM :
  • DOM 文档对象模型， 定义了一套操作HTML文档的API
  • BOM 浏览器对象模型，定义了一套操作浏览器窗口的API
  • window对象是一个全局对象，也可以说是JavaScript中的顶级对象
  • 像document、alert()、console.log()这些都是window的属性，基本BOM的属性和方法都是window的
  • 所有通过var定义在全局作用域中的变量、函数都会变成window对象的属性和方法
  • window对象下的属性和方法调用的时候可以省略window
  1. 定时器-延迟函数
  • 执行一次
  • setTimeout(回调函数, 延迟时间)
  • 清除延时函数：clearTimeout(timerId)
  • 清除定时函数：clearInterval(timerId)
  1. 事件循环（event loop）
  • 单线程：，所有任务需要排队如果js执行时间长，会造成页面渲染不连贯
  • 同步和异步：
    • 同步任务:同步任务都在主线程上执行，形成一个执行栈。
    • 异步任务:J5 的异步是通过回调函数实现的，一般而言，异步任务有以下三种类型:
      1. 普通事件，如 click、resize 等
      2. 资源加载，如 load、error 等
      3. 定时器，包括 setlnterval、setTimeout等
        异步任务相关添加到任务队列中(任务队列也称为消息队列)
  • 执行机制：
    • 先执行执行栈中的同任务
    • 异步任务放入任务队列
    • 执行栈执行完成就去任务队列执行，以此循环

  1. 正则表达式

  • 定义规则

    • 其中/ /是正则表达式字面量
    • 正则表达式也是对象

  • 使用正则

    • test()方法 用来查看正则表达式与指定的字符串是否匹配
    • 如果正则表达式与指定的字符串匹配 ，返回true，否则false

  • 元字符

    • 只能输入英文26个英文字母，换成元字符写法： /[a-z]/
    • 匹配开头的位置 ^：/^web/
    • 匹配结束的位置 $：/web$/
    • 精确匹配 ^ $:/^web$/

  • 量词

      • 重复次数 >= 0 次: /^w*$/
      • 重复次数 >= 1 次：/^w+$/
    • ? 重复次数 0 || 1 ：/^w?$/
    • {n} 重复 n 次：/^w{3}$/
    • {n,} 重复次数 >= n： /^w{2,}$/
    • {n,m} n =< 重复次数 <= m:/^w{2,4}$/

  • 范围

    • [abc] 匹配包含的单个字符
    • 想要包含小写字母，大写字母 ，数字 const reg3 = /^[a-zA-Z0-9]$/
    • 用户名可以输入英文字母，数字，可以加下划线，要求 6~16位
      const reg4 = /^[a-zA-Z0-9_]{6,16}$/
    • [^a-z] 取反符:只要不包含a-z都为true const reg5 = /^[^a-z]$/

  • 字符类

  • 修饰符

    • i 是单词 ignore 的缩写，正则匹配时字母不区分大小写
    • g 是单词 global 的缩写，匹配所有满足正则表达式的结果
    • replace const strEnd = str.replace(/前端/g, 'web')

day3

  1. 作用域
  • 局部作用域
    • 函数作用域
    • 块作用域：大括号内的
      • let和const会产生块作用域，var不会
      • 不同代码块的变量无法互相访问
  • 全局作用域：script标签或者js文件的最外层
  • 作用域链：函数执行时，优先查找当前函数作用域中的变量，找不到则逐级向上直到全局。
    • 子作用域能访问父，父无法访问子
  1. 垃圾回收机制
    1. 声明周期
      1. 内存分配：声明函数变量时
      2. 内存使用：读写内存，比如使用变量
      3. 内存回收：使用完毕，由垃圾回收器自动回收
    2. 全局变量一般不回收，除非页面关闭
    3. 局部变量的值，不用了会被自动回收掉
    4. 内存泄漏：程序中分配的内存由于某种原因未释放或者无法释放
    5. 算法
      1. 引用计数（IE）
        1. 问题：嵌套引用，两个对象互相引用，导致引用计数无法回收，内存泄漏
      2. 标记清除法（常用）
        1. 能从根部到达的对象就是有使用的，标记，否则清除
  2. 闭包
    1. 闭包=内层函数+外层函数的变量
    2. 用途：可以通过定义内部函数来让外面访问内部函数的变量
    3. 实现数据私用，防止篡改
    4. 内存泄漏的风险
  3. 变量提升
    1. 变量未声明就访问会语法报错
    2. 存在var时，系统会先把var变量提升到当前作用域最前面，然后执行代码，因此不管在哪声明var，都不会报错，会报undefined
    3. 只提升声明，不提升赋值
  4. 函数进阶
    1. 函数提升：函数声明前调用
      1. 会把所有函数声明提升到当前作用域最前面
      2. 函数表达式必须先声明和赋值再调用
  5. 函数参数
    1. 动态参数：arguments:伪数组 ，在函数内部直接用，不用访问。
    2. 剩余参数（析构）：getsum(…arr) 真数组
    3. 展开运算符：数组展开
  6. 箭头函数
    1. 箭头函数没有arguments只有剩余参数
    2. 箭头函数的this指向上一层作用域的this，对象不算作用域，逐级往上找直到window
    3. 普通函数的this指向它的调用者，如果普通函数嵌套箭头函数，那么箭头函数的this是普通函数的this
  7. 数组解构
    1. const [a,b,c]=arr
    2. 应用:交换变量 ;[b,a]=[a,b],前面要加一个分号
    3. 加分号的情况：立即执行函数和数组解构
    4. 利用剩余参数解决变量少，单元值多的情况 const [a,b,…c]=arr
    5. 防止undefined：可以默认传参 const [a=0,b=0]=arr
    6. 跳过某个参数：逗号隔开
  8. 对象解构
    1. 变量名和属性名必须一致，找不到相同就是undefined： const {uname,age}={uname:xm,age:18}
    2. 重新改名：const {uname:username,age}={uname:xm,age:18}
    3. 解构数组对象 const [{uname,age}]=pig
    4. 多级对象解构：注意对象名： const {name,family:{mother,father,sister}}=pig
  9. forEach
    1. 语法：arr.forEach(function(item,index){consoloe.log(item,index)})
    2. 不返回对象，适合遍历数组对象
  10. 对象
    1. 方法：
      1. let o ={}, const o =new obj()
    2. 构造函数
      1. new关键字调用函数的行为叫做实例化 const peppa = new pig(‘佩奇’)
      2. 返回的是新创建的对象
    3. 实例成员：实例对象的方法和属性，实例对象互相独立。
    4. 静态成员：构造函数中的方法和属性
  11. 内置构造函数
    1. 基础数据类型有专门的构造函数，所以有length等方法
    2. Object构造方法
    3. 获取属性名返回数组 ：Object.keys(o)
    4. 获取值返回数组：Object.values(o)
    5. 对象拷贝:Object.assign(obj,o) === o=>obj,可以用于添加属性
  12. Array
    1. reduce：total=arr.reduce(function(pre,cur){return pre+cur},begin)
    2. join:数组元素拼接为字符串
    3. find:返回满足条件的第一个元素，否则返回undefined: re=arr.find(item => item.name==='xm'})
    4. every(如果是空数组，会返回true),some:每个都是和有一个就行
    5. from:把伪数组转换为真数组，Array.from(lis)
  13. String
    1. split:把字符串转换为数组
    2. substring(begin,end)字符串截取
    3. startwith(‘xxx’,5)第五位开始，以xxx开头
    4. endwith
    5. includes,包含
