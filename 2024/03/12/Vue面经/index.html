1.Promise

promise解决异步操作的优点：

  • 链式操作减低了编码难度
  • 代码可读性明显增强

状态

  1. pending（进行中）
  2. fulfilled（已成功）
  3. rejected（已失败）

特点：状态只变一次

用法
Promise对象是一个构造函数，用来生成Promise实例

const promise = new Promise(function(resolve, reject) {});
Promise构造函数接受一个函数作为参数，该函数的两个参数分别是resolve和reject

resolve函数的作用是，将Promise对象的状态从“未完成”变为“成功”
reject函数的作用是，将Promise对象的状态从“未完成”变为“失败”

方法

  1. all
    1. 创建promise数组，const p = Promise.all([p1, p2, p3]);
    2. 数组中的每个元素都成功才算成功
    3. 如果数组中的元素自定义了catch那么不会触发all的catch
  2. race
    1. 创建promise数组,const p = Promise.race([p1, p2, p3]);
    2. p的状态与数组中最先改变的状态一致
  3. allSettled
    1. 接受数组
    2. 等到都返回结果才结束
  4. resolve
    1. 现有对象转为promise对象
      Promise.resolve('foo')// 等价于new Promise(resolve => resolve('foo'))
    2. 会执行对象的then方法
  5. reject
    1. 返回一个promise对象

2.SPA

SPA（single-page application），翻译过来就是单页应用,它通过动态重写当前页面来与用户交互，这种方法避免了页面之间切换打断用户体验

优点：

具有桌面应用的即时性、网站的可移植性和可访问性
用户体验好、快，内容的改变不需要重新加载整个页面
良好的前后端分离，分工更明确
缺点：

不利于搜索引擎的抓取
首次渲染速度相对较慢

3. vue生命周期

生命周期 描述 功能
beforeCreate 组件实例被创建之初 创建Vue的实例对象，此时只有内置函数
created 组件实例已经完全创建 此时自定义的data和方法初始化完成
beforeMount 组件挂载之前 此时将模版字符串渲染到内存dom中不执行
mounted 组件挂载到实例上去之后 vue实例进入运行阶段
beforeUpdate 组件数据发生变化，更新之前
updated 组件数据更新之后
beforeDestroy 组件实例销毁之前 销毁前， 可用于一些定时器或订阅的取消
destroyed 组件实例销毁之后
activated keep-alive 缓存的组件激活时
deactivated keep-alive 缓存的组件停用时调用
errorCaptured 捕获一个来自子孙组件的错误时被调用

4. Vue2和Vue3响应式原理的区别

  1. Vue2用的是defineproperty,vue3用的是proxy。

  2. defineProperty只能监听对象已有属性的访问和赋值，无法监控到数组方法，导致通过数组添加元素，不能实时响应。

  3. defineProperty只能劫持对象的属性，从而需要对每个对象，每个属性进行遍历,如果，属性值是对象，还需要深度遍历。

  4. vue3用的Proxy可以劫持整个对象，并返回一个新的对象。Proxy不仅可以代理对象，还可以代理数组。还可以代理动态增加的属性。
