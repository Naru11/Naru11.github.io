1.数据类型

按照存储位置的不同，分为基本类型和复杂类型

1. 基本类型：

  • Number
  • String：修改时会先销毁再创建
  • Boolean
  • Undefined: 声明变量没有初始化
  • null：null 值表示一个空对象指针，这也是给typeof传一个 null 会返回 “object” 的原因
  • symbol：符号的用途是确保对象属性使用唯一标识符，不会发生属性冲突的危险
    • 应用场景：可以用来当做key，以及保存私有属性 私有属性，外界不能直接访问

在数值类型中，存在一个特殊数值NaN，意为“不是数值”，用于表示本来要返回数值的操作失败了（而不是抛出错误）

2. 引用类型：复杂类型统称为object

  1. Object
  2. Array
    1. 判断方法：
      1. 使用 Array.isArray() 方法：console.log(Array.isArray(arr));
      2. 使用 instanceof 操作符：arr instanceof Array
      3. 使用 Array.prototype.constructor 属性：arr.constructor === Array
  3. Function
  4. Map 是一种键值对的集合,每个键唯一且与对应的值相关联
    1. 键可以是任意类型的值，包括原始类型和对象。
    2. 保持插入顺序，即按照元素插入的顺序进行迭代。
    3. 可以使用 Map.prototype.set() 方法添加键值对，使用 Map.prototype.get() 方法获取值，使用 Map.prototype.has() 方法检查键是否存在，使用 Map.prototype.delete() 方法删除键值对等
  5. Set 是一种值的集合，其中每个值都是唯一的，不会重复出现。与数组不同，Set 中的元素是无序的。
    1. 值可以是任意类型的值，包括原始类型和对象。
    2. 不允许重复的值，即相同的值只能出现一次。
    3. 可以使用 Set.prototype.add() 方法添加值，使用 Set.prototype.has() 方法检查值是否存在，使用 Set.prototype.delete() 方法删除值等。

3. 区别：

  1. 基本数据类型存储在栈中，引用类型的对象存储于堆中
  2. 引用类型对应的值存储在堆中，在栈中存放的是指向堆内存的地址
  3. 简单类型赋值，是生成相同的值，两个对象对应不同的地址
  4. 复杂类型赋值，是将保存对象的内存地址赋值给另一个变量。也就是两个变量指向堆内存中同一个对象

2. 数组的常用方法

1. 操作方法

  1. 增加

    1. 在原数组上修改，会对原数组产生影响
      1. push() ：末尾加
      2. unshift() ：数组最前面加
      3. splice() :可以指定位置 传入三个参数，分别是开始位置、0（要删除的元素数量）、插入的元素: colors.splice(1,0,'yellow')
    2. 不会产生影响 concat():首先会创建一个当前数组的副本，然后再把它的参数添加到副本末尾，最后返回这个新构建的数组，不会影响原始数组

  2. 删除

    1. 影响原数组：
      1. pop()：删除最后一项
      2. shift()：删除第一项
      3. splice()：传入两个参数，分别是开始位置，删除元素的数量，返回包含删除元素的数组
    2. 不影响：
      1. slice():用于创建一个包含原有数组中一个或多个元素的新数组，不会影响原始数组

  3. 查找：

    1. indexOf():下标 返回要查找的元素在数组中的位置，如果没找到则返回 -1

    2. includes():返回要查找的元素在数组中的位置，找到返回true，否则false

    3. find():返回第一个匹配的元素

2. 排序方法

  1. reverse:数组反转，在原数组上修改
  2. sort在原数组上修改，排序

3. 转换方法

  1. join():接受一个分隔符，分割数组，返回一个字符串：

  2. tostring() :变成字符串

4. 迭代方法

  1. 都不改变原数组
    1. some():对数组每一项都运行传入的测试函数，如果至少有1个元素返回 true ，则这个方法返回 true
    2. every():对数组每一项都运行传入的测试函数，如果所有元素都返回 true ，则这个方法返回 true
    3. forEach():对数组每一项都运行传入的函数，没有返回值
    4. filter:对数组每一项都运行传入的函数，函数返回 true 的项会组成数组之后返回
    5. map:对数组每一项都运行传入的函数，返回由每次函数调用的结果构成的数组
    6. reduce:累加

3. String的常用方法

1. 操作方法

  1. 增加：
    1. +和${}进行字符串的拼接
    2. concat ：let result = stringValue.concat("world");
  2. 删减:slice()substr()substring()
  3. 修改
    1. trim()、trimLeft()、trimRight(): 删除前、后或前后所有空格符，再返回新的字符串 let trimmedStringValue = stringValue.trim();
    2. repeat(): 接收一个整数参数，表示要将字符串复制多少次，然后返回拼接所有副本后的结果 let copyResult = stringValue.repeat(2) // na na
    3. padend(): 复制字符串，如果小于指定长度，则在相应一边填充字符，直至满足长度条件 console.log(stringValue.padStart(9, ".")); // "......foo"
    4. 大小写转换：toLowerCase()、 toUpperCase()
  4. 查找
    1. chatAt():传下标找值message.charAt(2); // "c"
    2. indexOf():传值找下标stringValue.indexOf("o")
    3. startWith()includes()

2. 转换方法

  1. split: 字符串转数组

3. 模板匹配方法

  1. match()
  2. serach()
  3. replace()

3. 类型转换

  1. 强制转换（显示）
    1. 其他转Number：1.
      1. 字符串中包含其他字符会返回NAN，
      2. true变1，false变0，
      3. undefined变NAN，null变0，
      4. 对象除了单个数值的数组正常，其他变为nan
    2. parseInt():遇到不能转换的字符停下：parseInt('32a3') //32
    3. 其他转String：都可以，对象需要注意，不会保存值
    4. boolen:除了false，空字符串，0，nan，undefined为false,其他都是true
  2. 自动转换（隐式）
    1. 需要布尔值的地方会自动转换
    2. 与字符串相加时会转换成字符串
    3. 其他运算符会转换成number，无法计算是nan ，undefined转换成数值是nan，

4. ==和===

  1. 等于运算符==
    1. 如果任一操作数是布尔值，则转换为数值，true==1
    2. 如果一个字符串一个数值，尝试转换数值，’55’==55
    3. null==undefined
    4. 只要一边是nan就返回false
    5. 如果是对象，则用valueof取原始值再比较
    6. 两个都为引用类型，则比较它们是否指向同一个对象
  2. 全等操作符
    1. 全等操作符由 3 个等于号（ === ）表示，只有两个操作数在不转换的前提下相等才返回 true。即类型相同，值也需相同

5. 深拷贝浅拷贝

  1. 浅拷贝：
    1. 问题：
      1. 对数组内引用类型的元素修改会共享，基本类型不会共享
    2. 如果拷贝的是基本数据类型，那么拷贝的是值，否则拷贝的是内存地址
    3. object.assign:var newObj = Object.assign({}, fxObj);
    4. slice:const fxArrs = fxArr.slice(0)
    5. 扩展运算符：const fxArrs = [...fxArr]
  2. 深拷贝：
    1. _.cloneDeep()：const obj2 = _.cloneDeep(obj1);
    2. jQuery.extend():const obj2 = $.extend(true, {}, obj1);
    3. JSON.stringify():const obj2=JSON.parse(JSON.stringify(obj1));
      1. 问题：会忽略undefined、symbol和函数
      2. 无法处理循环引用
      3. 丢失函数和原型信息
      4. 性能问题

6. 闭包

  1. 定义：内部函数访问到外部变量
  2. 使用场景：
    1. 创建私有变量：外界不能随意访问
    2. 延长变量的声明周期：比如某个函数执行完之后，函数里的变量也会销毁
  3. 问题：内存泄漏的风险

7. 作用域链

  • 局部作用域

    • 函数作用域
    • 块作用域：大括号内的
      • let和const会产生块作用域，var不会
      • 不同代码块的变量无法互相访问

  • 全局作用域：script标签或者js文件的最外层

  • 作用域链：函数执行时，优先查找当前函数作用域中的变量，找不到则逐级向上直到全局。

    • 子作用域能访问父，父无法访问子

  • 作用域链
    当在Javascript中使用一个变量的时候，首先Javascript引擎会尝试在当前作用域下去寻找该变量，如果没找到，再到它的上层作用域寻找，以此类推直到找到该变量或是已经到了全局作用域

9. this对象

严格模式下，不能将全局对象用于默认绑定，this会绑定到undefined，只有函数运行在非严格模式下，默认绑定才能绑定到全局对象

10.typeof与instanceof都是判断数据类型的方法，区别如下：

typeof会返回一个变量的基本类型，instanceof返回的是一个布尔值

instanceof 可以准确地判断复杂引用数据类型，但是不能正确判断基础数据类型

而typeof 也存在弊端，它虽然可以判断基础数据类型（null 除外），但是引用数据类型中，除了function 类型以外，其他的也无法判断
